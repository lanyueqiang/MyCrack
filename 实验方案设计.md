# 实验方案详细设计

## 一、数据集

### 1.1 主数据集：自采集机场跑道裂缝数据集（Airport Runway Crack Dataset, ARCD）

| 属性 | 详情 |
|------|------|
| 图像数量 | 2000 张 |
| 原始分辨率 | 1800 × 900 |
| 标注类型 | 像素级语义分割 mask |
| 来源 | 机场跑道实地采集 |

**数据划分**（固定随机种子，保证可复现）：

| 集合 | 数量 | 比例 |
|------|------|------|
| 训练集 | 1400 | 70% |
| 验证集 | 300 | 15% |
| 测试集 | 300 | 15% |

**预处理与增强**：
- 由于原始图像 1800×900 远大于网络输入尺寸，采用**随机裁剪 512×512** 作为训练输入
- 测试时使用**滑窗裁剪 + 拼接**，步长 256（50% overlap），避免边缘效应
- 训练数据增强：
  - 随机水平/垂直翻转（p=0.5）
  - 随机旋转 90°/180°/270°
  - 随机亮度/对比度抖动（±0.2）
  - 随机高斯模糊（σ=0~1.5）

**数据集描述要点（论文中需写）**：
- 说明采集设备、环境条件（光照、天气）
- 统计裂缝类型分布（横向/纵向/网状/块状各占多少）
- 统计裂缝宽度分布（≤2px, 2~5px, 5~10px, >10px）
- 统计裂缝像素占比（前后景比例，说明类别不平衡的严重程度）
- 展示典型样本图

### 1.2 公开数据集（用于泛化性验证）

#### (1) Crack500
| 属性 | 详情 |
|------|------|
| 图像数量 | 500 张（训练集有更多裁剪） |
| 分辨率 | 2000×1500（裁剪为 640×360 子图，共约 3000 张） |
| 来源 | 美国天普大学校园路面 |
| 标注 | 像素级 mask |
| 引用 | Yang et al., FPHBN, 2019 |

**选择理由**：裂缝分割领域最常用的benchmark之一，几乎所有近年论文都在此数据集上做了对比。

#### (2) DeepCrack
| 属性 | 详情 |
|------|------|
| 图像数量 | 537 张（训练 300 + 测试 237） |
| 分辨率 | 544×384 |
| 来源 | 多种路面和墙体 |
| 标注 | 像素级 mask |
| 引用 | Liu et al., DeepCrack, 2019 |

**选择理由**：包含多种表面类型的裂缝，适合验证跨场景泛化能力。

#### (3) CFD (Crack Forest Dataset)
| 属性 | 详情 |
|------|------|
| 图像数量 | 118 张 |
| 分辨率 | 480×320 |
| 来源 | 城市道路 |
| 标注 | 像素级 mask |
| 引用 | Shi et al., 2016 |

**选择理由**：经典小数据集，很多论文用它做对比，引用覆盖面广。

### 1.3 数据集使用策略

| 实验类型 | 使用数据集 | 说明 |
|---------|-----------|------|
| 主实验（对比+消融） | ARCD | 在自有数据集上充分验证 |
| 泛化性实验A | Crack500 | 在ARCD上训练，在Crack500测试集上直接测试（zero-shot） |
| 泛化性实验B | DeepCrack | 在ARCD上训练，在DeepCrack测试集上直接测试 |
| 泛化性实验C | CFD | 在ARCD上训练，在CFD上直接测试 |
| 补充实验 | Crack500 | 在Crack500上从头训练和测试，与文献结果直接对比 |

> **注意**：泛化性实验A/B/C是用ARCD训练的模型直接跨数据集测试，不做fine-tune。
> 补充实验是在该数据集上重新训练，目的是与现有论文的报告数据直接可比。

## 二、对比方法

### 2.1 通用语义分割方法

| 方法 | 年份 | 类别 | 选择理由 |
|------|------|------|---------|
| **U-Net** | 2015 | 编解码网络 | 最经典的分割baseline，也是我们的backbone基础 |
| **DeepLabV3+** | 2018 | 空洞卷积 | ASPP多尺度特征提取的代表性方法 |
| **HRNet** | 2020 | 高分辨率网络 | 始终保持高分辨率特征，适合精细分割 |
| **SegFormer** | 2021 | Transformer | 代表Transformer在分割任务上的SOTA |

### 2.2 裂缝检测专用方法

| 方法 | 年份 | 类别 | 选择理由 |
|------|------|------|---------|
| **DeepCrack** | 2019 | 多尺度特征融合 | 经典裂缝检测深度学习方法 |
| **FPHBN** | 2019 | 特征金字塔+层次提升 | Crack500数据集作者提出的方法 |
| **CrackFormer** | 2022 | Transformer+裂缝专用 | 近年裂缝检测领域代表性工作 |
| **CT-CrackSeg** | 2023 | CNN-Transformer混合 | 近两年的SOTA方法之一 |

### 2.3 对比方法实现方式

| 获取方式 | 方法 |
|---------|------|
| 官方开源代码复现 | DeepCrack, CrackFormer, FPHBN |
| 成熟框架实现 | U-Net, DeepLabV3+, HRNet, SegFormer (使用 mmsegmentation 或自行实现) |
| 论文复现 | CT-CrackSeg (若无开源代码) |

> **公平性原则**：所有方法统一使用相同的数据划分、数据增强、输入尺寸（512×512）、
> 训练epoch数上限和早停策略，保证对比公平。

## 三、评价指标

### 3.1 主要指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **Precision (P)** | TP / (TP + FP) | 预测为裂缝的像素中，真正是裂缝的比例 |
| **Recall (R)** | TP / (TP + FN) | 实际裂缝像素中，被正确检出的比例 |
| **F1-Score** | 2PR / (P + R) | 精确率和召回率的调和平均 |
| **mIoU** | mean(IoU_bg, IoU_crack) | 背景和裂缝两类的平均IoU |
| **IoU_crack** | TP / (TP + FP + FN) | 单独的裂缝类IoU（最关键指标） |

### 3.2 辅助指标

| 指标 | 说明 |
|------|------|
| **ODS** | 在所有图像上找到一个最优阈值的F值 |
| **OIS** | 每张图像各自找最优阈值的F值 |
| **AUC** | P-R曲线下面积 |
| **Params (M)** | 模型参数量（百万） |
| **FLOPs (G)** | 计算量（十亿浮点运算） |
| **FPS** | 推理速度（帧/秒，在固定GPU上测） |

> **指标选择理由**：
> - F1/IoU_crack 是裂缝分割论文最通用的核心指标
> - ODS/OIS 是边缘检测类论文（如BSDS500风格）的常见指标，部分裂缝论文会报告
> - Params/FLOPs/FPS 用于模型效率分析，回应审稿人可能关心的实用性问题

## 四、消融实验设计

### 4.1 核心消融：模块逐步叠加

验证每个创新模块的独立贡献和累积效果。

| 编号 | 配置 | VAFE | DGDC | IPWL | 说明 |
|------|------|------|------|------|------|
| M0 | Baseline | ✗ | ✗ | ✗ | U-Net + ResNet50 + BCE+Dice Loss |
| M1 | +VAFE | ✔ | ✗ | ✗ | 仅加灰度谷特征增强 |
| M2 | +DGDC | ✗ | ✔ | ✗ | 仅加方向引导可变形卷积 |
| M3 | +IPWL | ✗ | ✗ | ✔ | 仅换灰度先验加权损失 |
| M4 | +VAFE+DGDC | ✔ | ✔ | ✗ | 两个结构模块组合 |
| M5 | +VAFE+IPWL | ✔ | ✗ | ✔ | 特征增强+损失改进 |
| M6 | +DGDC+IPWL | ✗ | ✔ | ✔ | 卷积改进+损失改进 |
| M7 | **Full Model** | ✔ | ✔ | ✔ | **CrackPriorNet 完整模型** |

> 这个设计可以分析：
> - M1/M2/M3 vs M0：每个模块的独立贡献
> - M4/M5/M6 vs M1/M2/M3：两两组合的协同效果
> - M7 vs M4/M5/M6：三者结合是否优于两两组合
> - 1+1是否>2 的协同性分析

### 4.2 VAFE 模块内部消融

| 编号 | 变体 | 说明 |
|------|------|------|
| V1 | 单方向Hessian (垂直) | 只用一个方向的谷响应 |
| V2 | 双方向Hessian (水平+垂直) | 两个正交方向 |
| V3 | **四方向Hessian (0°/45°/90°/135°)** | 完整四方向（默认配置） |
| V4 | 融合位置：Stage1 | 在编码器第1层融合 |
| V5 | **融合位置：Stage1+Stage2** | 浅层两级融合（默认配置） |
| V6 | 融合位置：Stage1+2+3 | 扩展到中间层 |

> 验证：方向数量的影响 + 最佳融合位置

### 4.3 DGDC 模块内部消融

| 编号 | 变体 | 说明 |
|------|------|------|
| D1 | 标准可变形卷积（无方向引导） | 对比方向引导的必要性 |
| D2 | 方向引导 + DCNv1 | 无采样点权重 |
| D3 | **方向引导 + DCNv2** | 有采样点权重（默认配置） |
| D4 | DGDC 用在 Stage3+4 | 仅在深层使用 |
| D5 | **DGDC 用在 Stage4+5** | 在最深两层使用（默认配置） |
| D6 | DGDC 用在 Stage3+4+5 | 全部深层使用 |

> 验证：方向引导 vs 自由偏移 + DCNv1 vs DCNv2 + 使用位置

### 4.4 IPWL 损失函数消融

| 编号 | 变体 | 说明 |
|------|------|------|
| L1 | BCE only | 纯二值交叉熵 |
| L2 | BCE + Dice | 标准组合损失（Baseline配置） |
| L3 | BCE + Dice + Focal | 加入Focal Loss处理不平衡 |
| L4 | BCE + Dice + 灰度权重（不含距离变换） | 仅用灰度值做权重 |
| L5 | BCE + Dice + 距离变换权重（不含灰度） | 仅用到骨架距离做权重 |
| L6 | **BCE + Dice + IPWL（灰度+距离）** | 完整灰度先验加权（默认配置） |

> 验证：IPWL中灰度先验和距离变换各自的贡献

### 4.5 超参敏感性分析

需要分析的关键超参数：

| 超参 | 搜索范围 | 说明 |
|------|---------|------|
| α (Dice Loss 权重) | {0.5, 1.0, 2.0, 5.0} | IPWL中各子损失的平衡 |
| β (IPWL 权重) | {0.1, 0.5, 1.0, 2.0, 5.0} | 灰度先验损失项的强度 |
| Hessian σ (高斯尺度) | {1.0, 2.0, 3.0, 5.0} | VAFE中Hessian计算的平滑尺度 |
| DGDC 采样点数 | {3×3=9, 5×5=25} | 可变形卷积核大小 |

## 六、实验实施细节（Implementation Details）

### 6.1 训练配置

| 配置项 | 设置 |
|--------|------|
| 框架 | PyTorch |
| GPU | 2 × NVIDIA RTX 3090 (24GB × 2) |
| 分布式训练 | PyTorch DDP（DistributedDataParallel） |
| Backbone | ResNet50 (ImageNet pretrained) |
| 优化器 | AdamW |
| 初始学习率 | 1e-4 |
| 学习率调度 | CosineAnnealingLR，T_max=最大epoch |
| 权重衰减 | 1e-4 |
| Batch size | 16（每卡8） |
| 最大训练epoch | 200 |
| 早停策略 | 验证集 IoU_crack 连续30个epoch不提升则停止 |
| 输入尺寸 | 512 × 512 |
| 随机种子 | 42（固定所有随机源） |

### 6.2 公平对比原则

所有对比方法统一以下设置：
- 相同的数据划分和数据增强
- 相同的输入尺寸 512×512
- 相同的优化器和学习率策略
- 相同的最大epoch和早停策略
- 使用各方法原论文推荐的损失函数（BCE+Dice 或其各自默认）
- 如原论文有推荐Backbone则使用原推荐，否则统一 ResNet50
- 每个实验跑3次取平均值±标准差
